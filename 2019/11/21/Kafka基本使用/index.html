1. Kafka基本概念

1.1 Kafka中的基本术语

Producer

消息生产者，往某个Topic上发布消息，也负责选择发布到Topic的哪一个分区。

Consumer

消息消费者，一个发布在Topic上的消息允许广播给多个消费者组，但是每个消费者组只能有一个消费者可消费该消息。

Broker

Kafka集群中的每个Kafka节点服务器。每一个Broker保存Topic的一个或多个Partitions

Topic

Kafka将消息进行分类，每一类的消息称之为一个Topic

Partition

Kafka下数据存储的基本单元，其提供了顺序保证和负载均衡。

  • 顺序保证：每个Partition仅被唯一一个消费者组中的一个消费者消费到，那么该消费者就是这个Partition的唯一消费者，并按顺序消费数据
  • 负载均衡：每个Topic有多个Partition，那就需要对多个消费者做负载均衡，注意Partiton数≥消费者数

1.2 Kafka基本架构

Kafka是强依赖于Zookeeper的。Kafka将Broker信息、Topic的Partition分布情况、以及消费者的offset信息（0.9版本之前）等存储在Zookeeper上。不过生产者不依赖Zookeeper。

进入Zookeeper查看Kafka的相关信息：

1.2.1 Broker
  • /brokers/ids/[0…N]：Broker的id
  • /brokers/topics/[topic]/partitions/[0…N]（其中[0..N]表示partition索引号）：存储某个topic的partitions所有分配信息
1.2.2 Consumer（老版本）

老版本是把offset信息存储在Zookeeper的，比如在使用控制台发送消息时，指定的是kafka-console-producer.sh --zookeeper。现在新版本是kafka-console-producer.sh --broker-list，存储在Broker上了：

1.3 Kafka数据存储

消息发送时都被送到一个Topic，其本质就是目录（地址由server.properties的log.dirs指定）；Topic是由指定数目的Partition Logs组成的（数目由server.properties的num.partitions指定），每个Partition Logs磁盘上对应一个文件夹，数据就存在00000000000000000000.log文件。

4. 使用Kafka Connect来导入/导出数据

Kafka Connect是导入和导出数据的一个工具。它是一个可扩展的工具，运行连接器，实现与自定义的逻辑的外部系统交互。具体可参考官方文档。

一些概念：

  • Connectors：通过管理task来协调数据流的高级抽象
  • Tasks：如何将数据复制到Kafka或从Kafka复制数据的实现
  • Workers：执行Connector和Task的运行进程

Kafka Connect的工作模式：

  • Standalone：单机模式，所有的worker都在一个独立的进程中完成
  • Distributed：分布式模式，在有效的worker进程中它们会自动去协调执行connector和task，不管新增或挂了一个worker，其它的worker都会自动重新分配connector和task。具有高扩展性以及自动容错机制。
