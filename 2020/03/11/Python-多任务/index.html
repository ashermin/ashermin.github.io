Python 多任务

发表于 2020-03-11 更新于 2020-03-16 分类于 Python 阅读次数：

线程（Thread）是操作系统能够进行运算调度的最小单位。

进程（Process）是程序执行的实例，也是资源的集合。

进程提供程序执行的所有资源：虚拟的地址空间、可执行的代码、操作系统的接口、安全的上下文（记录启动该进程的用户和权限等等）、唯一的进程ID、环境变量、优先级类、最小和最大的工作空间（内存空间），还要有至少一个线程。

1.4 守护线程

setDaemon(True)把线程设置为主线程的守护线程，当主程序结束后，子线程即使没有执行完毕，也会随之马上结束。

由于子线程需要等待1秒，所以当主线程运行完就直接结束了。

1.5 互斥锁

就是上厕所，一个坑位只能进一个人。互斥锁可以避免产生脏数据。

1.6 递归锁

递归锁的用法和互斥锁一样。区别是它的内部维护着一个Lock和一个counter变量，counter记录了acquire()的次数，直到某一个线程的acquire()和release()的次数一样了后，才会释放资源。

1.7 信号量

互斥锁同时只允许一个线程更改数据，而Semaphore是同时允许一定数量的线程更改数据 。比如厕所有3个坑，那最多只允许3个人上厕所，后面的人只能等里面有人出来了才能再进去。

1.8 事件 Event()

事件用于主线程控制其它线程的执行，是一个简单的线程同步对象。

事件处理的机制：全局定义了一个Flag，当Flag值为False，那么event.wait()就会阻塞；当Flag值为True，那么event.wait()便不再阻塞。

方法 注释
clear 将flag设置为“False”
set 将flag设置为“True”
is_set 判断是否设置了flag
wait 会一直监听flag，如果没有检测到flag就一直处于阻塞状态

2. 多进程

每个进程都是由父进程提供的。每启动一个子进程就从父进程克隆一份数据，但是进程之间的数据本身是不能共享的。

os.getppid()获取父进程号，os.getpid()获取自己的进程号：

2.1 进程间的通信

进程之间数据是不共享的，多进程之间的通信通过Queue()或Pipe()来实现。

Queue()与Pipe()相类似，都是先进先出的结构。但Queue()允许多个进程放入，多个进程从队列取出对象；Pipe()只能两个线程。

2.1.1 Queue

Queue()，消息队列，FIFO形式，单向通道

2.1.2 Pipe

Pipe()`，消息管道，FIFO形式，双向管道

返回2个连接对象(conn1, conn2)，代表管道的两端，每端都有send()和recv()方法。

默认是双向通道，可以通过mutiprocessing.Pipe(duplex=False)创建单向管道，那么返回的对象conn1只负责接收消息，conn2只负责发送消息。

⚠️注意：如果两个进程试图在同一时间的同一端进行读取和写入，那么这可能会损坏管道中的数据。

2.2 进程池

初始化时可以指定最大进程数，如果进程池序列中没有可供使用的进程，那么程序就会等待，直到进程池中有可用进程为止。

进程池中常用方法：

  • apply() 同步执行（串行）
  • apply_async() 异步执行（并行）
  • terminate() 立刻关闭进程池
  • join() 主进程等待所有子进程执行完毕。必须在close()或terminate()之后
  • close() 等待所有进程结束后，才关闭进程池

2.3 使用进程池拷贝文件

需求：使用进程池，只开3个进程来对文件夹的文件进行拷贝，并且使用multiprocessing.Manager()实现主进程与进程池之间的通信，现实拷贝的进度。

3. 协程

3.1 迭代器

当使用一个循环（比如 for 循环）来遍历容器（比如列表，元组）中的元素时，这种遍历的过程就叫迭代。

迭代器是指遵循迭代器协议的对象。

迭代器协议（iterator protocol）是指要实现对象的 __iter()__ 和__next__() 方法。其中：

  • __iter__() 方法返回迭代器对象本身
  • __next__() 方法返回容器的下一个元素，在没有后续元素时抛出StopIteration 异常

案例1：自定义Subject类，让这个类可以在for循环中打出课程名字

案例2：使用迭代器的方式实现斐波那契数列

使用迭代器实现的方式好处在于不会直接生成结果，而是一个一个地执行代码，大大节省了内存空间。

3.2 生成器

所谓生成器就是一边循环一边计算的机制，定义上为：如果一个函数定义中包含yield关键字，那这个函数就是一个生成器（generator）。生成器是一种特殊的迭代器。

---1---只会跑一次，当程序运行到---2---后，通过yield返回b值，然后程序卡住等待；直到再次调用__next__()时，程序才会继续执行，打印---3---，然后再走一次while循环。

值得注意的是，当生成器里的元素都取完了后，如果再调用next()来取，将会报StopIteration错，解决的方法就是抛异常：

send()

send()和next()都可以让生成器代码从暂停到继续执行，但是next()只能得到yield后面的值，而send()可以将yield b设置为发送的新值：

send()发送的数据会传递给yield b，即ret。传递完后程序继续执行，直到又运行到yield b后，返回b的值。
