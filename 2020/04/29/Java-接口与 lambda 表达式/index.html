Java 接口与 lambda 表达式

发表于 2020-04-29 更新于 2020-05-11 分类于 Java 阅读次数：

接口

实现接口的关键字——implements，比抽象类还要抽象的纯抽象接口，不能有实例字段，但可以有静态字段。

接口的特性

（1）接口可以被别的接口所继承

（2）接口不能包含实例域或静态方法，但可包含常量

接口中的方法都自动为public，接口中的域自动设为public static final。

接口和抽象类

抽象类有点类似接口，但是抽象类只能被extends，所以只能继承一个，接口却可以implements无数个。

默认方法

接口的实现类需要强迫实现默认的抽象方法；但是default方法不会强迫实现类实现它，default方法可以不重写，但是如果接口中的默认方法不能满足某个实现类需要，那么实现类可以重写默认方法。

默认方法冲突情况：先在一个接口中定义了一个默认方法，然后在超类或另一个接口中定义了同样的方法。

解决：

  1. 接口冲突。如果两个接口的默认方法同名，实现类同时实现了这两个接口，那必须在实现类中重写默认方法；
  2. 优先超类。会执行超类的默认方法，接口中的同名同参数方法会被忽略。

⚠️注意：千万不能重新定义 Object 类中的某个方法，如toString()、equals()等。

静态方法

静态方法只能通过接口名来调用，不能通过实现类的类名/对象来调用；而default类必须通过实现类的对象调用。

内部类

内部类主要有四类：成员内部类、局部内部类、匿名内部类和静态内部类。

成员内部类

在另一个类的内部，可以无条件访问外部类的所有成员属性和方法，包括private和static。由于它是寄生在外部类的，所以如果要创建成员内部类，必须先创建一个外部类的对象。

不过外部类要访问成员内部类的属性或方法，就需要先创建一个内部类对象，再通过这个对象的引用来访问。

内部类拥有的权限若是public，哪里都能访问；若是protected，只能在同一包下或继承外部类的情况；若是default，只能在同一包下访问；若内部类是private，则只能在外部类访问。

局部内部类

局部内部类是创建在方法里的，可以看作是方法里的一个局部变量，它的访问权限仅限于这个方法内，所以不能有public、protected、private和static修饰。

匿名内部类

匿名内部类就是没有名字的内部类，其作用是继承其他类或是实现接口，不需要新增方法，仅仅对继承方法的实现或重写。

静态内部类

静态内部类声明关键字是static，它是为了把自己这个内部类隐藏在一个外部类里，并不需要内部类引用外部类对象。所以当内部类不需要访问外部类对象时，应该使用静态内部类。

静态内部类不能使用外部类的非static变量或方法，因为外部类的非静态变量或方法需要新建外部类对象才能创建，而静态内部类可以在没有外部类对象的情况下直接创建。
