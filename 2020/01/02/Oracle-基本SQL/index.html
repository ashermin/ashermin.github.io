1. 常用数据类型

加粗的表示常用。

1.1 字符型

字符型在数据库中是以ASCII码的格式存储的：

数据类型 取值范围(字节) 说明
varchar2 0~4000 可变长度的字符串（最常用），与char类型区别就是它不会用空格填充至最大长度
nvarchar2 0~1000 包含unicode格式数据的变长字符串
char 0~2000 不可变长，会用空格填充至最大长度
nchar 0~1000 Unicode字符集的定长数据（适合中文数据）
long 0~2G 变长的字符串（基本不用，尽可能将long类型转换为clob类型）

1.2 数字型

数据类型 取值范围 说明
number(p,s) p最大精度是38位（十进制） p代表的是有效数字，s代表的是小数点后的位数。可以用来存储定长的整数和小数（常用）
NUMBER(5,2)：有效数字5位，保留2位小数，如123.45
float 用来存储126位数据（二进制） 存储精度是按二进制计算的，精度范围为二进制的1~126，在转化为十进制时需要乘以0.30103

1.3 日期类型

timestamp与date类型的区别为：timestamp显示日期和时间更精确：

数据类型 说明
date 日期和时间，精确到秒。范围：公元前4712年1月1日~公元9999年12月31日
timestamp 日期和时间，精确到小数点后9位，显示上午/下午

1.4 其它类型

还有存放大数据量的数据类型和二进制文件的数据类型：

数据类型 取值范围(字节) 说明
blob 最多可以存放4GB 存储二进制数据，适合存储电子表格、字处理文档、图像文件等（不过一般不会使用blob类型直接存储图片、文档的二进制数据，往往是保存它在磁盘上的路径）
clob 最多可以存放4GB 存储字符串数据，适合纯文本
bfile 大小与操作系统有关 这种数据类型允许在数据库列中存储一个Oracle目录对象（操作系统目录的一个指针）和一个文件名，并读取这个文件

NULL值的各种处理

排序：排序时默认NULL值为最大值，即升序在最后降序在首位。不过可以手动指定NULL值的位置：

查询：检索NULL数据不能使用=，而要使用IS NULL或IS NOT NULL

四则运算：对NULL进行四则运算后，其值都为NULL，这时就要用到NVL或NVL2单行函数

  • NVL(x, value)：若x为NULL，则返回value；否则返回x
  • NVL2(x, value1, value2)：若x为NULL，则返回value1；否则返回value2
模糊查询——LIKE

通配符：_表示一个字符，%表示0到多个字符

范围查询
  • BETWEEN a AND b：闭区间[a, b]
  • IN / NOT IN：从列表里选取
  • SOME / ANY / ALL：子查询时配合<、>和=来使用
    • ANY：满足子查询结果的任意一个
    • SOME：和ANY用法一样，不过ANY多用在非“=”环境
    • ALL：满足子查询所有结果
GROUP BY和HAVING

⚠️注意：使用GROUP BY时，SELECT列表中只能存在分组函数或GROUP BY子句中的字段。

HAVING和GROUP BY联合使用，其与WHERE的区别就是WHERE是与单个值有关，而HAVING与组有关。换句话说就是，HAVING对于GROUP BY创建的组负责；WHERE对FROM负责。

6.2 字符型函数

ASCII码与字符串转换

ASCII码➡️字符串：

字符串➡️ASCII码：

拼接两个字符串——CONCAT(c1, c2)或||
搜索字符的位置——INSTR(c1, c2, [i], [j])和INSTB(c1, c2, [i], [j])

这两个函数区别是对汉字、全角符的计算：INSTR按1个算，而INSTRB按3个算

  • c1：被搜索的字符串
  • c2：希望搜索的字符串
  • i：搜索的开始位置，默认为1
  • j：第j次出现的位置，默认为1

如果value=ZHS16GBK，那么一个汉字占用2个字节；如果value=AL32UTF8，那么一个汉字占用3个字节：

所以本篇笔记中，我的中文字占3个字节。

字符串长度
  • LENGTH(x)：以字符为单位
  • LENGTHB(x)：以byte字节为单位（由于我是AL32UTF8，所以中文我这里占3个字节）
  • LENGTHC(x)：以Unicode字符为单位
  • LENGTH2(x)：以UCS2代码点为单位
  • LENGTH4(x)：以UCS4代码点为单位
替换字符串——REPLACE(c1, c2, [c3])

功能：将c1中的c2字符串用c3替换。c3不写即为将c2从c3中删除

截取字符串——SUBSTR(c1, n1, [n2])

  • n1：字符串开始位置（从1开始）

    • n1=0：从首字母开始，等效于n1=1
    • n1<0：从末尾开始

  • n2：截取的位数（省略表示从n1开始一直到最后）

首尾去除字符串

TRIM的用法与LTRIM、RTRIM不一样，不过它们c2不写都默认为去除空格：

  • TRIM(c1 [FROM c2])：左右两边去除
  • LTRIM(c2, c1)：左边去除
  • RTRIM(c2, c1)：右边去除
字母大小写转换

整个字符串大小写转换：LOWER(c1)和UPPER(c1)

每个单词首字母大写：INITCAP(c1)

6.3 日期型函数

系统的日期时间

SYSDATE：系统时间，一般与TO_CHAR()转换函数联用

SYSTIMESTAMP：系统时间带时区

系统的时区

DBTIMEZONE：返回时区

当前会话的时区、日期、时间

CURRENT_TIMESTAMP：当前会话的时间，带时区

CURRENT_DATE：当前会话的时间

日期操作

ADD_MONTHS(date, integer)：在制定月份上添加月份

LAST_DAY(date)：指定日期对应月份的最后一天

NEXT_DAY(date, char)：如果char值是’星期一’，则返回date指定日期的下个星期一是哪天

MONTH_BETWEEN(date1,date2)：日期date1到日期date2之间的月数

EXTRACT(fmt FROM date)：提取日期中的特定部分。fmt为YEAR、MONTH、DAY、HOUR、MINUTE和SECOND

6.4 转换函数

日期➡️字符串：TO_CHAR(date, [fmt], [params])

  • fmt：转换格式（YY YYYY YEAR｜MM MONTH｜DD DAY｜HH24 HH12｜MI｜SS）

  • params：控制格式化时使用的语言类型，一般不写

字符串➡️日期：TO_DATE(char, [fmt], [params])

⚠️注意：TO_DATE()是转回为系统默认日期格式，所以很少用！

数字➡️字符：TO_CHAR(number, [fmt])
  • 9：显示数字并忽略前面的0
  • 0：显示数字，位数不足，用0补齐
  • .或D：显示小数点
  • ,或G：显示千位符
  • $：美元符号
  • S：加正负号（前后都可以）
字符➡️数字：TO_NUMBER(char, [fmt])

6.5 NULL函数

排除指定条件函数：LNNVL(condition)

该函数可以得到除了condition要求条件之外的数据（包括NULL），通常用在WHERE条件中。

替换NULL值

NVL(x, value)：如果x为NULL，则返回value；否则返回x

NVL2(x, value1, value2)：如果x为NULL，则返回value1；否则返回value2

6.6 集合函数

集合函数通常配合GROUP BY或HAVING使用，除了COUNT函数其它都会忽略列值为NULL的数据。

求平均值：AVG([DISTINCT] field)
记录数量：COUNT(* | [DISTINCT] field)
最大/最小值：MAX([DISTINCT] field)/MIN([DISTINCT] field)
求和：SUM([DISTINCT] field)

6.7 分析函数

排序：RANK()和DENSE_RANK()

区别：

  • RANK()是跳跃排序，有两个第二名时接下来就是第四名，1,2,2,4

  • DENSE_RANK()是连续排序，有两个第二名时后面仍然为第三名，1,2,2,3

语法：RANK() OVER(PARTITION BY field1 ORDER BY field2 [DESC])

顺序：ROW_NUMBER()

语法：ROW_NUMBER() OVER (PARTITION BY field1 ORDER BY field2 [DESC])
