1. PL/SQL结构和符号

1.1 基本结构

声明部分：变量和常量的定义。该部分可选

执行部分：执行部分是 PL/SQL 块的指令部分，由关键字BEGIN开始，关键字END结尾。所有的可执行 PL/SQL 语句都放在这一部分，该部分执行命令并操作变量。其它的 PL/SQL 块可以作为子块嵌套在该部分。该部分是必选的。

异常处理部分：该部分用EXCEPTION关键字把可执行部分分成两个小部分，之前的程序是正常运行的程序，一旦出现异常就跳转到异常部分执行。该部分是可选的

2. PL/SQL的变/常量和数据类型

2.1 常量和变量的声明

2.1.1 常量声明

常量在声明时赋予初值，并且在运行时不允许重新赋值。使用CONSTANT关键字声明常量，使用:=或DEFAULT赋值

打开控制台的输出显示功能：SET SERVEROUTPUT ON;

在 SQLPlus 中还可以声明 Session（会话，也就是一个客户端从连接到退出的过程称为当前用户的会话。）全局级变量，该变量在整个会话过程中均起作用，类似的这种变量称为*宿主变量**。宿主变量在 PL/SQL 引用时要用:变量名引用

2.1.2 变量声明

语法：变量名 数据类型[:=初始值]

变量的值可以直接给一个定值，也可以从表中字段获取。

例1：直接给定一个值

例2：从表中获取值，并对值做异常处理，输出到终端

SELECT...INTO...是PL/SQL特有的语法，作用：把SELECT查询出来的值一一对应存储到INTO后的变量名中。每次只能接受返回1条记录，超过1条或没有返回记录会引发异常。

2.2 数据类型

2.2.1 标量数据类型

标量数据类型的变量只有一个值，且内部没有分量。标量数据类型包括数字型，字符型，日期型和布尔型。这些类型有的是 Oracle SQL 中定义的数据类型，有的是 PL/SQL 自身附加的数据类型。字符型和数字型又有子类型，子类型只与限定的范围有关，比如 NUMBER 类型可以表示整数，也可以表示小数，而其子类型 POSITIVE 只表示正整数。

2.2.2 属性数据类型

当声明一个变量的值是数据库中的一行或者是数据库中某列时，可以直接使用属性类型来声明。Oracle中存在两种属性类型：%ROWTYPE和%TYPE

（1）%ROWTYPE：以数据库表中的一行作为数据类型，可以使用.来访问记录中的属性

（2）%TYPE：引用某个变量或者数据库的列的类型作为某变量的数据类型

3. PL/SQL控制语句

3.1 顺序控制

实现顺序控制主要是靠GOTO和NULL这两个语句：

  • GOTO语句将无条件的跳转到标签指定的语句去执行。标签是用双尖括号括起来的标示符<<XXX>>，在 PL/SQL 块中必须具有唯一的名称，标签后必须紧跟可执行语句或者 PL/SQL 块

⚠️注意：GOTO 不能跳转到 IF 语句、CASE 语句、LOOP 语句、或者子块中。

  • NULL语句什么都不做，只是将控制权转到下一行语句。NULL 语句是可执行语句。NULL语句在 IF 或者其他语句语法要求至少需要一条可执行语句，但又不需要具体操作的地方。比如 GOTO 的目标地方不需要执行任何语句时

3.2 条件控制

PL/SQL 中关于条件控制的关键字有 IF-THEN、IF-THEN-ELSE、IF-THEN-ELSIF 和多分枝条件 CASE。

3.2.1 IF-THEN

例：查询 JAMES 的工资，如果大于 900 元，则发奖金 800 元

3.2.2 IF-THEN-ELSE和IF-THEN-ELSIF

例：查询 JAMES 的工资，如果大于 1500 元，则发放奖金 100 元，如果工作大于900元，则发奖金 800 元，否则发奖金 400 元

3.2.3 CASE

例：键盘输入一个字母 ，如果是A、B、C 就输出对应的级别信息；如果不是就打出GG

3.3 循环控制

  • LOOP无条件循环
  • WHILE循环
  • FOR循环

上面的三类循环中EXIT用来强制结束循环，相当于 break。

3.3.1 LOOP无条件循环

在 LOOP 循环中可以使用EXIT或者EXIT WHEN 条件的形式终止循环。否则该循环就是死循环

例：求1+2+3+…+99+100的值

3.3.2 WHILE循环

其实WHILE循环就是在LOOP无条件循环前面加个了WHILE条件

例：求1+2+3+…+99+100的值

3.3.3 FOR循环

其实FOR循环就是在LOOP无条件循环前面加个了FOR条件

REVERSE：让循环方式与默认的相反，加了就从循环上限➡️循环下限。

⚠️注意：循环计数器增量（或减量）必须为1。

例：求1+2+3+…+99+100的值

4. PL/SQL的DML和DDL

DML语句在PL/SQL中的使用和单独执行DML操作没有区别；而SELECT和DDL的使用都有点区别。

4.1 DML的SELECT

对于SELECT的区别其实就是PL/SQL特有的SELECT...INTO...，案例可以参考“2.1.2 变量声明”的例2。

4.2 DDL

DDL要在PL/SQL中使用需要执行EXECUTE IMMEDIATE命令。

例：动态创建表Emp_test

⚠️注意：不建议在PL/SQL中使用DDL，通常是在存储过程中执行DDL语句。

5. PL/SQL异常

ORACLE异常分为两种类型：系统异常、自定义异常。其中系统异常又分为：预定义异常和非预定义异常

  • 系统异常
    • 预定义异常：Oracle预定义的异常情况大约有24个。对这种异常情况的处理，无需在程序中定义，由Oracle自动将其引发
    • 非预定义异常：即其它标准的Oracle错误。对这种异常情况的处理，需要用户在程序中定义，然后由Oracle自动将其引发
  • 自定义异常：程序执行过程中，出现编程人员认为的非正常情况。对这种异常情况的处理，需要用户在程序中定义，然后显式地在程序中将其引发

5.1 预定义异常

对于Oracle常见的一些错误，已经定义好了它们的错误编号和异常名字。在对这种异常情况的处理时，只需在PL/SQL块的异常处理部分，直接引用相应的异常情况名，并对其完成相应的异常错误处理即可。具体有哪24个可以参见附录表1：预定义异常。

6. PL/SQL函数

6.1 创建函数

语法：

参数说明：

  • [OR REPLACE]：覆盖同名函数。一般只有确认function_name函数是新函数或是要更新的函数时，才使用OR REPALCE关键字，否则容易删除有用的函数

  • IN | OUT | IN OUT：形参的模式。若省略，则为IN模式

    • IN：形参只能将实参传递给形参，进入函数内部，但只能读不能写，函数返回时实参的值不变
    • OUT：形参会忽略调用时的实参值（或说该形参的初始值总是NULL），但在函数内部可以被读或写，函数返回时形参的值会赋予给实参
    • IN OUT：具有前两种模式的特性。即调用时，实参的值总是传递给形参；结束时，形参的值传递给实参

    ⚠️注意：调用时，对于IN模式的实参可以是常量或变量；但对于OUT和IN OUT模式的实参必须是变量。

  • {IS | AS}：二选一。该项后面是PL/SQL块

例：创建一个类似MAX(x, y)的函数。如果之前该函数已存在，则添加OR REPLACE关键字
